Transition

The transition property allows you to change the style of an element smoothly over a specified duration. For example, you can change the background color of a button when a user hovers over it.

Example:

            
            .button {
                background-color: Blue;
                transition: Background-color 0.5s ease;
                }
                
            .button:hover {
                    background-color: green;
                }

Keyframes animation

Keyframes allow you to define a series of styles that an element should have at certain points in time. You can then use these keyframes to create an animation.

Example:

            
                @keyframes slidein {
                    from {
                        transform: translateX(-100%);
                    }
                    to {
                    transform: translateX(0);
                    }
                    }
                    
                    .slide {
                        animation: slidein 1s ease;
                    }

Transform

The transform property allows you to modify the position, size, and shape of an element. You can use transforms to create animations such as scaling, rotating, and skewing.

Example

            
                .box {
                    transform: scale(1);
                    transition: transform 0.5s ease;
                }
                
                .box:hover {
                    transform: scale(1.2);
                }

-Intersection Observer API

This is JavaScript code that uses the Intersection Observer API to toggle a CSS class on and off for each article element when it becomes visible or invisible in the veiwport.

Here's what the code does step by step:

  1. The code selects all article elements on the page using the 'querySelectorAll' method and assigns them to a variable called 'articles'.
  2. It creates and empty object called 'options'. This object can be used to configure the observer with additional options, but in this case, no options are set.
  3. It creates a new IntersectionObserver object calles 'entries' and the 'observer' object itself. The 'entries' array contains one entry for each observed element that has intersected or stopped intersecting the veiwportsince the last time the callback was called.
  4. For each entry in the 'entries' array, the code checks whenether it is intersecting the veiwport by checking the 'isIntersecting' property of the entry object. If it is not intersecting, the function simply returns and does nothing. If it is intersecting, the function toggles the 'visible' class on the 'target' property of the entry object, which refers to the observed element itself.
  5. The code then loops over each article element in the 'articles' array and calls the 'observe' method of the 'observer' object, passing in the article element as an argument. This tells the observer to watch for changes in the visibility of each article element.

Example:

            
                const articles = document.querySelectorAll("article");
                const options = {};
                const observer = new IntersectionObserver(function 
                (entries, observer) {
                  entries.forEach((entry) => {
                    if (!entry.isIntersecting) {
                      return;
                    }    entry.target.classList.toggle("visible");
                  });
                }, options);
                articles.forEach((article) => {
                  observer.observe(article);
                });

Overall, this code allows you to create a "lazy loading" effect for your article elements, where they are only loaded and displayed on the page when they are actually visible in the veiwport. This can improve page load times and performance by reducing the amount of content that needs to be loaded and rendered at once.
